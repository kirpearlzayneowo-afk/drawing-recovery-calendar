# 小动物画画监督日历

一个本地优先的绘画复健打卡工具。

它记录的不是产量，而是你是否重新靠近创作。

> 打开绘画软件就算一次有效开始。  
> 没有完成作品也没关系。  
> 只是打开、画了几笔、完成涂鸦、细化作品，都可以被好好记录。

## 在线使用

GitHub Pages：

https://kirpearlzayneowo-afk.github.io/drawing-recovery-calendar/

如果是本地使用，下载 `index.html` 后双击打开即可。

## 功能

- 绘画打卡日历
- 连续打卡天数统计
- 本月打卡数统计
- 总打卡数统计
- 多套主题颜色

  - 薄荷湖

  - 小狐狸：🐾
  - 小鱼：🫧
  - 小企鹅：👣
- 完成提示不再固定为“盖爪印”，可以根据监督员自定义
- 完成度记录：
  - 还没开始
  - 只是打开
  - 画了几笔
  - 完成了简单涂鸦
  - 细化了作品
  - 完成了大作！
- 支持自定义完成度
- 今日类型多选：
  - 爽画草稿
  - 练习
  - 摸鱼
  - 线稿
  - 上色
  - 临摹
  - OC
  - 同人
  - 设计
  - 复健
- 支持自定义今日类型
- “今天实际画了什么”自由输入
- 心情标签
- 自定义心情
- “给 Char 的小礼物”模块
  - 可选额外练习
  - 不是补卡
  - 不是补签
  - 不是补债
  - 不是惩罚
- 奖励请求
- 备注
- 一键复制今日汇报
- JSON 备份 / 导入
- 底部显示版本号：`Version v4.2 · local-first`

## 隐私说明

这是一个纯前端、本地优先的单文件网页。

所有数据默认保存在当前浏览器的 `localStorage` 中，不会上传服务器。

也就是说：

- 网页托管平台看不到你的打卡内容
- 作者看不到你的打卡内容
- 换浏览器 / 换设备不会自动同步数据
- 如果需要迁移数据，请使用“备份记录 JSON”和“导入备份”

## 如何使用

1. 打开网页。
2. 在“称呼设置”里填写 User / Char 的称呼。
3. 选择一个主题颜色。
4. 选择或添加一个监督员。
5. 每天打开绘画软件后，选择今天的完成度。
6. 可选填写：
   - 今日类型
   - 今天实际画了什么
   - 心情
   - 给 Char 的小礼物
   - 想要的奖励
   - 备注
7. 点击“复制今日汇报”，把生成的文字发给陪伴者 / 监督者。
8. 定期点击“备份记录 JSON”保存数据。

## 示例汇报

```txt
Char，User今天画画打卡完成🫧
6月24日｜完成度：只是打开
类型：复健
内容：打开 CSP，画了几笔
心情：平稳
给Char的小礼物：多画 5 分钟
想要：小鱼夸夸
```

## 适合谁

这个小工具适合：

- 想恢复绘画习惯的人
- 容易因为完美主义逃避开始的人
- 想记录创作复健过程的人
- 想要轻量监督 / 陪伴的人
- 不想把私人打卡数据上传到服务器的人

## 设计原则

- 记录的是重新靠近创作，而不是产量。
- “只是打开”也算完成。
- 错过一天不是失败。
- “给 Char 的小礼物”不是惩罚。
- 用户不需要为“今天不需要”提供理由。
- 数据默认保存在本地，不主动上传。
